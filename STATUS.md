### Update code on VPS
```bash
cd /opt/seolab
git pull
docker compose up -d --build backend worker beat   # rebuild + recreate (don't forget --build!)
```

### Restart only (env-only changes, code unchanged)
```bash
docker compose restart backend worker beat
```

### View logs
```bash
docker compose logs -f backend       # live
docker compose logs backend --tail 100 | grep -B 2 -A 20 Traceback   # errors
```

### Postgres shell
```bash
docker compose exec db psql -U seolab seolab
```

### Backup database (manual)
```bash
docker compose exec -T db pg_dump -U seolab seolab > backup-$(date +%Y%m%d).sql
```

### Trigger background tasks manually
```bash
# Run audit for a project (in a Celery container, not backend)
docker compose exec worker celery -A workers.celery_app.celery call workers.tasks.run_site_audit_for_project --args '["PROJECT_UUID"]'

# Run rank tracker
docker compose exec worker celery -A workers.celery_app.celery call workers.tasks.run_rank_tracker_for_all_projects
```

## Architecture recap

```
Browser (HTTPS)
      ↓
Caddy (host, port 80/443) ── auto Let's Encrypt
      ↓
Frontend container (Nginx, port 8080) ── serves Vite SPA + proxies /api
      ↓
Backend container (FastAPI/uvicorn, port 8000)
      ↓ ↘
   Postgres   Redis (broker + cache, 24h TTL on external API responses)
              ↑
         Celery worker (concurrency=4) + Celery beat (cron schedules)
                  ↑
                External APIs: DataForSEO · GSC · Claude · PageSpeed · Open PageRank · Google Suggest
```

All container-to-container traffic on the `seolab` Docker bridge network. Only port 80 + 443 exposed on host (UFW).
